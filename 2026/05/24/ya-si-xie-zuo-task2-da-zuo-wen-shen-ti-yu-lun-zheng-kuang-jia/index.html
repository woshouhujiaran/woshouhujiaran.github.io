雅思写作 Task 2：大作文审题与论证框架

发表于2026-05-24|更新于2026-05-24|分享
|浏览量:

一、Task 2 本质先想清楚

大作文本质是：以议论文为载体，陈述可被普遍认可的社会观点。评分重点不是“观点多新”，而是“是否切题、是否论证到位、是否表达清楚”。

写作前先做两件事：

  1. 识别题型（结果判断 / 命题判断 / 分析报告）。
  2. 确定范围（谁、在什么时期、做什么事、影响到哪里）。

二、结果判断类（比较优缺点、两件事好坏）

这类题要回答“结果是好是坏”，核心是 4 个定位点：

  1. action：题目中的行为是什么，不要扩大或缩小定义域。
  2. target：这个行为作用在谁身上。
  3. effect：最终形成的结果是什么（好/坏）。
  4. scope：题目是否限定了人群、时间、场景，论证必须扣这个范围。

段落展开模板（实用版）

可按 观点句 + condition + action + effect 展开：

  1. 观点句：target 通过某个 action 获得（或失去）某个 desired effect。
  2. condition：指出阻碍/帮助目标达成结果的现实条件。
  3. action：解释该行为如何解决问题（或加剧问题）。
  4. effect：说明最后如何以可观察的形式体现为好/坏结果。

可选强化：

  1. 加 common 例证，提高说服力。
  2. 在同一维度做正反 action 对比，避免跑题。

双边结构怎么选

如果正反都能展开，优先双边：

  1. 平衡式：正反笔墨接近。
  2. 让步式：先正后反（或先反后正）。
  3. 让步拉回：先承认对方一点，再回到自己主张并反驳。

注意：让步拉回必须在同一评价维度反驳，不要“换维度偷换论题”。

三、命题判断类（证明命题对错）

这类题属于“事实判断”，常见问法是 To what extent do you agree?。

建议按 4 步走：

  1. statement：命题具体是什么。
  2. facts：用普遍事实（或反例）支撑。
  3. verdict：给出对/错或部分同意结论。
  4. scope：明确结论生效范围。

使用规则：

  1. 要证明“对”，尽量给社会普遍存在的事实。
  2. 要证明“错”，至少给一个能成立的反例。
  3. 双边题可对正反两边各跑一遍 4 步。

四、分析报告类（原因 + 解决方案）

这类题通常问：现象为何出现、如何解决。结构天然是双边展开。

推荐骨架：Issue -> Cause -> (Effect) -> Solution -> Scope

  1. Issue：重述现象与问题。
  2. Cause：拆内部原因与外部原因（社会、经济、科技、文化）。
  3. Effect：可选，用来过渡到对策。
  4. Solution：方案要直面根源，并说明个人/社会/政府各做什么。
  5. Scope：最后收束适用范围与优先级。

五、单边与双边的选择原则

  1. 单边：至少 2 个分论点，写得顺可上 3 个。
  2. 双边：适合比较题，尤其 dual 类型题目。
  3. 选择标准：哪种结构更容易“充分回应题干限制”，就选哪种。

六、段落层面的通用论证工具

  1. condition：说明前提条件。
  2. action：给出行为或措施（可配例证）。
  3. fact：用普遍事实或可验证现象兜底。

例证建议：

  1. 优先 common、可共识的例子。
  2. 分类例证可从不同方面支持同一论点。
  3. 个人故事能用，但要改写成“普适经验”，避免过度私人化。

七、开头和结尾怎么处理

开头

  1. 先复述题目背景/争议点。
  2. 再明确立场。
  3. 立场必须由分论点支撑，不能空喊观点。

结尾

  1. 可不强求花哨。
  2. 作用是总结总论点与分论点，不新增观点和例子。
  3. 建议控制在 50 词以内。

八、评分标准对应写法（重点保底）

1. TR（Task Response）

  1. 观点、立场、理由必须互相匹配。
  2. 回答题目每个子问题，且回应充分。
  3. 题干限制（范围、对象、时态）要体现完整。
  4. 分论点必须扩展，并有理由或例子支撑。

2. CC（Coherence and Cohesion）

  1. 常用 4-5 段，总分总结构清楚。
  2. 段内建议：中心句 + 支撑 +（可选）结论句。
  3. 连接词要精准，不滥用。
  4. 适当用指代减少重复，但避免指代模糊。

3. LR（Lexical Resource）

  1. 用词以准确为先，再谈“高级”。
  2. 可适度冒险使用书面表达，但不夸张、不失真。

4. GR（Grammar Range and Accuracy）

  1. 稳定使用复杂句型（让步、条件、定语从句等）。
  2. 允许少量错误，但不能影响理解。
  3. 标点要规范，尤其逗号和冒号。

九、考场执行流程（可直接照做）

  1. 审题 2-3 分钟：圈出 action、target、scope。
  2. 定立场 1 分钟：先有分论点，再下总立场。
  3. 列提纲 3 分钟：每段写“结果句 + 原因/事实 + 例子”。
  4. 写作 25-28 分钟：优先完成主体段论证。
  5. 检查 3-5 分钟：重点查跑题、主谓一致、时态、指代。

这套框架的核心不是“背模板”，而是每段都回答同一个问题：这个 action 到底是在帮助，还是在阻碍我们想要的 desired effect。
