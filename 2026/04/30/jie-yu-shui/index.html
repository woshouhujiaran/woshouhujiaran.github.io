力扣题解：接雨水

题目链接

点击查看题目

解题思路

一句话总结：先分别预处理每个位置左侧最高柱子和右侧最高柱子，再按 min(左高, 右高) - 当前高度 累加每一格能接的水。

代码执行流程如下：

  1. 先拿到数组长度 n，如果 n == 0 直接返回 0，避免后续访问越界。
  2. 创建 leftmax 数组，leftmax[i] 表示从左到右到达 i 位置时见过的最高柱子。
  • leftmax[0] = height[0]
  • 从 i = 1 到 n-1，递推 leftmax[i] = max(leftmax[i-1], height[i])
  1. 创建 rightmax 数组，rightmax[i] 表示从右到左到达 i 位置时见过的最高柱子。
  • rightmax[n-1] = height[n-1]
  • 从 i = n-2 到 0，递推 rightmax[i] = max(rightmax[i+1], height[i])
  1. 遍历每个位置 i，该位置最多能蓄水的高度上限是两侧较矮的一边：min(leftmax[i], rightmax[i])。
  • 所以当前位置蓄水量为 min(leftmax[i], rightmax[i]) - height[i]
  • 把每个位置结果累加到 ans。
  1. 返回总蓄水量 ans。

关键设计点：

  • 用两个预处理数组把“每个位置左右最高值”的查询降为 O(1)，从而只需线性遍历一次求总和。
  • min(leftmax[i], rightmax[i]) 体现木桶短板效应，水位由较低的一侧决定。

复杂度分析

时间复杂度

O(n)。

原因：

  • 一次循环构建 leftmax
  • 一次循环构建 rightmax
  • 一次循环统计答案

总计是常数次线性扫描，仍为 O(n)。

空间复杂度

O(n)。

原因：额外使用了 leftmax 和 rightmax 两个长度为 n 的数组，辅助空间线性增长。
