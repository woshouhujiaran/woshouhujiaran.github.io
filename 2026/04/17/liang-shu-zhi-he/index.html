力扣题解：两数之和

点击查看题目

解题思路

这段代码的核心思路是：一边遍历数组，一边用哈希表记录已经出现过的数字和它的下标，同时检查当前数字需要的配对值是否已经出现。

  1. 先定义哈希表 mp，类型是 unordered_map<int, int>。
    作用是记录“数字 -> 下标”的映射，表示这个数字在前面出现过。

  2. 用 for 循环从左到右遍历 nums，当前下标为 i。
    每次先计算 x = target - nums[i]，x 表示当前数字还需要的另一个值。

  3. 通过 mp.count(x) 判断 x 是否已经在前面出现。
    如果出现，直接返回 {mp[x], i}，分别是之前那个数的下标和当前下标。

  4. 如果没找到，就把当前数字存入哈希表：mp[nums[i]] = i，供后续元素匹配。

  5. 最后 return vector<int>(); 作为兜底返回。

关键设计点：

  • 先查找再插入，避免当前元素和自己配对。
  • 哈希表把“查找某个数是否出现过”从线性优化到均摊常数时间。

复杂度分析

时间复杂度

O(n)。

数组只遍历一遍，循环 n 次。每次哈希查找/插入均摊 O(1)，所以总时间复杂度是 O(n)。

空间复杂度

O(n)。

额外使用哈希表存储遍历过的元素，最坏情况下需要存近 n 个键值对。
