力扣题解：两数之和

题目链接

点击查看题目

解题思路

这段代码的核心思路是：一边遍历数组，一边用哈希表记录已经出现过的数字和它的下标，同时检查当前数字需要的“配对值”是否已经出现。

  1. 先定义一个哈希表 mp，类型是 unordered_map<int, int>。
    作用是记录“数字 -> 下标”的映射，表示这个数字之前在数组中出现过。

  2. 用 for 循环从左到右遍历 nums，当前下标是 i。
    每次取到当前值 nums[i]，先计算：
    x = target - nums[i]。
    这里的 x 表示“如果要凑成 target，当前数字还需要哪个数”。

  3. 用 mp.count(x) 判断这个“需要的数”之前有没有出现过。
    如果出现过，说明已经找到了两个数：

  • 之前出现的那个数下标是 mp[x]
  • 当前数下标是 i
    然后直接返回 {mp[x], i}。

  1. 如果没找到配对值，就把当前数字加入哈希表：
    mp[nums[i]] = i。
    表示后续元素可以和当前元素做匹配。

  2. 代码最后有 return vector<int>();。
    在 LeetCode 这题的前提下通常一定有解，所以这行主要是让函数结构完整。

关键设计点：

  • 先查找再插入，避免把当前元素和自己配对。
  • 哈希表把“查某个数是否出现过”从线性查找降到近似常数时间。

复杂度分析

时间复杂度

O(n)。

数组只遍历一遍，循环执行 n 次。每次循环中的哈希表查询和插入平均是 O(1)，所以总时间复杂度是 O(n)。

空间复杂度

O(n)。

使用了额外的哈希表 mp 来存储遍历过的元素，最坏情况下会存下接近 n 个键值对，所以空间复杂度是 O(n)。
